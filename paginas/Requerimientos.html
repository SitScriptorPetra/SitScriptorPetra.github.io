Tecnica de recopilacion de requerimientos

Técnica de recopilación de requerimientos: Utilizamos la técnica “Brainstorm” o Lluvia de ideas para encontrar los requerimientos del sistema. Cada uno de los in tegrantes integró los requerimientos que creyó convenientes y con ayuda de retroalimentación pudimos expresar la idea de cada una de una mejor manera.

No. Requerimiento clasificación
1 Si la información requerida por los usuarios no se encuentra dentro del sistema, que la interfaz bote un mensaje diciendo que no ha encontrado la información solicitada RF
2 Las búsquedas deberán ser precisas, esto es que cualquier falta de ortografía o palabra mal escrita no obtendrá ningún tipo de resultado RX
3 El sistema habilita al locutor para que pueda modificar, consultar, agregar, editar o eliminar algún tipo de dato en el programa RF
4 El acceso al sistema se da a través de interfaz web RX
5 Los álbumes de un cantante se ordenan de forma descendente por su fecha de lanzamiento. RF
6 Es sistema debe mostrar las tablaturas de canciones en caso de que las haya RF
7 El sistema hace búsquedas en la base de datos de manera eficiente, si el usuario ingresa la cadena de búsqueda de manera correcta, con una respuesta de 3s. RNF
8 Para mostrar una imagen el sistema accede al nombre de la dirección de carpeta en la base de datos, la cual nos da la ruta a un almacenamiento físico, esto para ahorrar desde 1.2 a 5 mb de espacio en la BD RNF
9 El sistema tiene una alta usabilidad, en donde las interfaces son ordenadas e intuitivas, de acuerdo a la satisfacción del cliente RNF
10 El sistema cuenta con un backup en la base datos, para que en caso de algún apagón o borrado de datos accidental. RF
11 Dada la optimizacion de codigo, el sistema cuenta con un rendimiento acelerado, puede inicializarse en un tiempo promedio de 2.5s. RNF
12 El sistema obtiene de la base de datos la información requerida para poder estructurar la página web RF
13 El sistema mantiene la integridad de la BD asegurando que las validaciones de los datos sean amplias RF
14 El sistema corre sobre un navegador que sea compatible como mínimo con el estándar ecmascript 2015 y posteriores RX
15 El sistema informa al usuario si ocurre algún error al hacer alguna operación en la base de datos RF
16 El sistema muestra sugerencias a la búsqueda hecha por el locutor RF
17 El sistema necesita una conexión a internet para funcionar adecuadamente RX
